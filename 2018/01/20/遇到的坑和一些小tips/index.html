                       三、
                       现象：golang环境变量配置&&go初探&&fish配置

                       解决：$ go build <packworkpath> 第一个文件名不能有'/', '/'是根目录的意思！！！！
                            要想使用$ go install <workpath> 需要正确配置下面的路径： $ vim ~/.config/fish/config.fish

                            1  set -x GOPATH $HOME/gowork
                            2 set -x PATH $PATH /usr/local/go/bin/ $GOPATH/bin

                                 新建了config.fish文件在其中加入如上两行，重启fish。
                                  如果在bash中命令应该在～/.bashrc最后一行添加：

                                      \$ export GOPATH=\$HOME/gowork
                                      \$ export PATH=\$PATH:$GOPATH/bin

                                      之后source .bashrc 就可以。如果在命令行中输入为临时变量，下次使用需要重新设置 在home目录下为本用户使用。

                                      注意：1、bash和fish的这两行命令是对应的，fish中一律使用set来设置。bash中使用':'作为两个路径的分隔符，即把\$PATH和后面的目录均赋值给PATH，\$PATH是系统设定的路径！！！忽略它会发生很可怕的事情QAQ 2、在fish中使用空格来分割不同路径，但记得把\$PATH写上。引入包时注意格式，上面设置的路径为根，go运行时不用写直接写下属的src之类的目录运行。

七、

问题：gitlab从远程仓库clone，更改后再将本地分支push到远程仓库的某一个分支。

方法：可以在本地新建同名分支—-git add . 增加更改—-git commit -m “blablabla” —-git push—-这个时候会有提示说远程上游分支不符合并且告诉更改上游的语句—-输入—重新git push

这个命令：git push –set-upstream origin master

参考：segmentfault git上游分支:https://segmentfault.com/q/1010000009915017
