《leetcode放弃指南》s48

發佈於 2018-02-26 leetcode 

题目：

You are given an n x n 2D matrix representing an image.

Rotate the image by 90 degrees (clockwise).

Follow up:
Could you do this in-place?

分析：

首先看变换前后行号和列号的对应关系，按照题目示例，题目要求为旋转90度，首先看第一列5,2,13,15,该列旋转后第一行为15,13,2,5，因此使用栈储存矩阵的列值，竖向入栈，横向出栈。有一个问题就是，在第一列出栈时，数值会被替换，这时候该如何有规律的储存替换值？
另一种考虑方法是，先将矩阵转置，如第一列转换后第一行为5,2,13,15,转置后再每行进行对称的变化。

目前还有bug难过，放弃。
转置过程里有一行数字不进行转置，是循环的逻辑有问题，当j跳出循环时，i还需要参与进去循环，也许是这里导致的。

输入：[[1,2,3],[4,5,6],[7,8,9]]
转置后输出：[[1, 4, 3], [2, 5, 8], [7, 6, 9]]
输出：[[3,4,1],[8,5,2],[9,6,7]]
应得答案：[[7,4,1],[8,5,2],[9,6,3]]

那么转置怎么写呢。

转职ok了，因为对python不了解，一开始没有用for，第一堆错了的原因是因为当i = 2时 j已经等于1了，不清零j的话，(2,0)(0,2)就无法置换。
那么接下来的问题是怎么把行调换呀？

倒转用的python切片，如a[0:3]指从索引0取到索引3,即0,1,2。倒数第一个元素的索引为-1 。a[-10:]为后十个数字，a[:]可以复制与原来相同的list。a[::2]从头到尾每隔两个取一个。
如果自己写数组倒置的话，用栈写。
